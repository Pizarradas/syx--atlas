06.2 — Densidad y flujo editorial (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema gestiona:

la cantidad de contenido visible
la distribución de bloques
el recorrido del usuario

No se trata de cuántas noticias hay.
Se trata de cómo se organizan para ser entendidas.

Principio base

Una portada no se llena.
Se equilibra.

1. Qué es la densidad editorial

Es la relación entre:

cantidad de contenido
espacio disponible
jerarquía
2. Qué es el flujo editorial

Es el recorrido que sigue el usuario al:

escanear
leer
navegar
3. Relación entre densidad y flujo
demasiada densidad → rompe el flujo
poca densidad → pierde interés

El sistema debe encontrar equilibrio.

4. Zonas de lectura

La portada debe estructurarse en zonas:

entrada (impacto)
desarrollo (exploración)
profundidad (detalle)
5. Regla de entrada

La parte superior debe:

captar atención
mostrar jerarquía
facilitar comprensión inmediata
6. Regla de desarrollo

La zona media debe:

ampliar contenido
ofrecer variedad
mantener ritmo
7. Regla de profundidad

Las zonas inferiores deben:

ofrecer más contenido
permitir exploración
no saturar
8. Alternancia de densidad

El sistema debe alternar:

bloques densos
bloques ligeros

Esto genera:

→ ritmo

9. Bloques de respiración

Espacios que permiten:

descansar visualmente
reorganizar atención
10. Agrupación editorial

Los contenidos deben organizarse en bloques:

coherentes
identificables
consistentes
11. Evitar saturación

Señales de saturación:

demasiados elementos visibles
falta de jerarquía
dificultad para decidir
12. Evitar vacío estructural

Señales de vacío:

falta de contenido relevante
espacios sin intención
13. Regla de progresión

El contenido debe:

empezar fuerte
mantenerse interesante
no decaer abruptamente
14. Ritmo vertical

El usuario debe poder:

recorrer la portada
sin fricción
sin interrupciones
15. Relación con jerarquía

La densidad debe respetar la jerarquía.

16. Relación con spacing

El spacing regula:

densidad
ritmo
flujo
17. Sistema de límites

Definir:

máximo de bloques por zona
densidad máxima
número de elementos visibles
18. Adaptación al contexto

En mobile:

reducir densidad
simplificar flujo
19. Evaluación del flujo

Preguntas clave:

¿qué ve primero el usuario?
¿qué hace después?
¿dónde se pierde?
20. Regla final

Si el usuario no sabe por dónde empezar:

→ el sistema ha fallado

Ejemplo simplificado

Portada:

arriba → noticia principal
medio → bloques secundarios
abajo → contenido extendido

↓

Resultado:

flujo claro

Cierre

El flujo no se diseña.

Se construye.
