04.1 — Proporciones y escalas (SYX Atlas)
Naturaleza de este documento

Este documento define cómo se construyen:

tamaños
relaciones
jerarquías
spacing

a partir de sistemas de proporción y escala.

Principio base

Nada crece de forma arbitraria.
Todo crece siguiendo una relación.

1. Sistema de escala base

Todo el sistema parte de una escala base.

Esta escala define:

tipografía
spacing
dimensiones
módulos
2. Tipos de escala

El sistema puede trabajar con diferentes tipos de escala, pero debe elegir una como base principal.

Escala modular (recomendada)

Ejemplo:

1
1.25
1.5
1.75
2
Escala basada en razón áurea (uso controlado)

Ejemplo:

1
1.618
2.618

Uso limitado para:

jerarquías fuertes
elementos clave
Escala lineal

Ejemplo:

4
8
12
16

Uso recomendado para:

spacing fino
ajustes menores
3. Regla de elección de escala

El sistema debe:

usar una escala principal
evitar mezclar múltiples escalas sin control
mantener consistencia global
4. Aplicación a tipografía

La tipografía debe seguir una escala.

Ejemplo conceptual
base → 1
cuerpo → 1.25
subtítulo → 1.5
titular → 2
5. Aplicación a spacing

El spacing debe derivar de la misma escala.

Evitar:

valores independientes
ajustes manuales
6. Aplicación a layout

Los bloques deben responder a proporciones.

Ejemplo:

columna principal → 2x
columna secundaria → 1x
7. Regla de consistencia

Una vez definida la escala:

→ todo el sistema debe usarla

8. Jerarquía proporcional

La jerarquía se construye mediante:

diferencias de tamaño
diferencias de espacio
diferencias de agrupación

No mediante estilos arbitrarios.

9. Sistema de ratios

El sistema puede definir ratios estándar:

1:1
1:1.25
1:1.5
1:2

Estos ratios deben reutilizarse en todo el sistema.

10. Escala adaptable

La escala no es fija en valores.

Es fija en relaciones.

11. Continuidad de escala

El sistema debe evitar saltos abruptos.

Debe existir progresión natural.

12. Control de extremos

Evitar:

escalas demasiado comprimidas
escalas demasiado abiertas
13. Relación entre sistemas

Tipografía, spacing y layout deben compartir lógica.

No pueden ser sistemas independientes.

14. Regla de simplificación

Menos niveles de escala → mayor claridad.

15. Sistema de mínimos y máximos

Definir:

tamaño mínimo legible
tamaño máximo razonable
spacing mínimo
spacing máximo
16. Proporción en componentes

Cada componente debe construirse proporcionalmente.

No mediante medidas fijas.

17. Escala y contexto

La escala debe adaptarse al contexto sin romper relaciones.

18. Regla de reutilización

Las proporciones deben reutilizarse en todo el sistema.

19. Validación de escala

Cada escala debe evaluarse según:

legibilidad
coherencia
adaptabilidad
20. Regla final

Si un valor no pertenece a la escala:

→ no debe usarse

Ejemplo simplificado

Escala base:

1
1.25
1.5
2

Aplicación:

texto → 1
subtítulo → 1.25
título → 1.5
hero → 2

Spacing:

base → 1
separación media → 1.5
separación grande → 2
Cierre

La escala define el comportamiento del sistema.

Sin escala, todo se vuelve arbitrario.
