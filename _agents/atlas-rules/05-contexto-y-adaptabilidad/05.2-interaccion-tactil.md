05.2 — Interacción adaptativa (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema se adapta a:

distintos tipos de interacción
distintos niveles de precisión
distintos contextos de uso

No se diseña solo para verse.
Se diseña para interactuar.

Principio base

La interfaz no es solo visual.
Es un sistema de interacción.

1. Tipos de interacción

El sistema debe contemplar:

táctil (dedo)
puntero (ratón/trackpad)
teclado
voz
interacción pasiva (solo consumo)
2. Variables de interacción

El sistema debe adaptarse a:

precisión
velocidad
intención
contexto de uso
3. Interacción táctil

Características:

baja precisión
contacto directo
uso rápido
Reglas
áreas de interacción amplias
separación suficiente entre elementos
feedback inmediato
4. Interacción con puntero

Características:

alta precisión
posibilidad de hover
mayor control
Reglas
interacciones más finas
estados adicionales (hover, focus)
menor necesidad de espacio
5. Interacción por teclado

Características:

navegación secuencial
sin control visual directo en algunos casos
Reglas
foco claro
orden lógico
accesibilidad estructural
6. Interacción por voz

Características:

sin interfaz visual directa
dependencia de comandos
Reglas
simplificación extrema
estructuras claras
lógica predecible
7. Interacción pasiva

El usuario no interactúa activamente.

Consume contenido.

Reglas
claridad máxima
jerarquía evidente
navegación implícita
8. Adaptación sin duplicación

El sistema no crea versiones distintas.

Adapta comportamiento dentro del mismo sistema.

9. Principio de accesibilidad

La accesibilidad no es un extra.

Es parte estructural del sistema.

10. Feedback del sistema

Toda interacción debe generar:

respuesta inmediata
confirmación clara
estado reconocible
11. Estados de interacción

El sistema debe definir:

normal
hover
active
focus
disabled

Todos coherentes.

12. Prevención de errores

El sistema debe:

evitar errores
facilitar corrección
no penalizar al usuario
13. Carga cognitiva en interacción

La interacción debe:

ser intuitiva
reducir decisiones
evitar complejidad innecesaria
14. Consistencia de comportamiento

La misma acción debe producir:

→ el mismo resultado

15. Jerarquía interactiva

No todos los elementos tienen el mismo nivel de interacción.

Debe existir:

prioridad
diferenciación
16. Ritmo de interacción

El sistema debe respetar:

velocidad del usuario
tiempo de respuesta
17. Interacción en contexto restrictivo

Ejemplo:

en movimiento
con una mano
con atención limitada

El sistema debe simplificar.

18. Integración con layout

La interacción debe estar alineada con:

jerarquía
spacing
estructura
19. Evaluación de interacción

Cada solución debe responder a:

¿es fácil de usar?
¿es rápida?
¿es clara?
20. Regla final

Si una interacción necesita explicación:

→ está mal diseñada

Ejemplo simplificado

Táctil:

botones grandes
separación clara

↓

Mouse:

interacción más precisa
hover disponible

↓

Sistema:

misma lógica
diferente comportamiento
Cierre

La interacción no es una capa.

Es parte del sistema.
