06.1 — Jerarquía de contenido (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema:

asigna importancia al contenido
organiza esa importancia
la traduce en estructura

No es una cuestión visual.
Es una cuestión de prioridad informativa.

Principio base

La jerarquía no se diseña.
Se calcula.

1. Qué es la jerarquía

Es el sistema que determina:

qué se ve primero
qué se ve después
qué puede ignorarse
2. Variables de jerarquía

La jerarquía debe construirse a partir de variables como:

relevancia
actualidad
interés
contexto
impacto
3. Peso informativo

Cada contenido tiene un peso.

Este peso determina su posición en el sistema.

4. Sistema de niveles

El sistema debe trabajar con niveles definidos:

Nivel 1 — Principal
foco absoluto
máxima visibilidad
Nivel 2 — Destacado
alta relevancia
visibilidad clara
Nivel 3 — Secundario
apoyo
contexto
Nivel 4 — Complementario
contenido adicional
bajo impacto
5. Regla de foco único

Solo puede existir:

→ un nivel principal

6. Distribución de niveles

El sistema debe limitar:

número de elementos por nivel
proporción entre niveles
7. Traducción a estructura

La jerarquía se traduce en:

tamaño
posición
spacing
agrupación
8. Regla de proporcionalidad

A mayor jerarquía:

→ mayor presencia estructural

9. Evitar igualdad jerárquica

No todos los contenidos pueden tener el mismo peso.

10. Contraste jerárquico

Debe existir diferencia clara entre niveles.

11. Jerarquía y layout

El layout debe reflejar la jerarquía.

No al revés.

12. Jerarquía y ritmo

La jerarquía define el ritmo de lectura.

13. Jerarquía y densidad

Más jerarquía → más espacio
Menos jerarquía → mayor densidad

14. Jerarquía dinámica

La jerarquía puede cambiar según:

contexto
momento
contenido
15. Regla de claridad

El usuario debe entender la jerarquía sin pensar.

16. Evitar sobrejerarquización

Demasiados niveles generan:

confusión
ruido
17. Sistema de límites

Definir:

máximo de niveles
máximo de elementos destacados
18. Validación jerárquica

Cada solución debe responder:

¿qué es lo primero que se ve?
¿qué es lo segundo?
¿qué se puede ignorar?
19. Señales de fallo
todo parece igual
múltiples focos
confusión
20. Regla final

Si la jerarquía no es evidente en segundos:

→ no está bien definida

Ejemplo simplificado

Portada:

1 noticia principal
3 destacadas
resto secundarias

↓

Resultado:

claridad inmediata

Cierre

La jerarquía no es estética.

Es la estructura del significado.
