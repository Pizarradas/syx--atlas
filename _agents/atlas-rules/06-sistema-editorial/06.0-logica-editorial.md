06 — Sistema editorial (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema organiza:

contenido editorial
jerarquía informativa
flujo de lectura
integración de bloques

No se trata de diseño visual.
Se trata de cómo se estructura la información para ser entendida.

Principio base

El sistema no muestra contenido.
Organiza significado.

1. Qué es el sistema editorial

Es el conjunto de reglas que determina:

qué contenido aparece
en qué orden
con qué jerarquía
con qué relación
2. El contenido como unidad base

El sistema trabaja con contenido, no con componentes.

Cada pieza de contenido tiene:

peso informativo
contexto
intención
3. Tipos de contenido

El sistema debe contemplar:

noticia principal
noticia secundaria
bloques de apoyo
contenido de contexto
contenido publicitario
4. Jerarquía editorial

La jerarquía no es visual.

Es informativa.

Debe responder a:

relevancia
actualidad
interés
5. Niveles de jerarquía

Ejemplo:

contenido principal
contenido destacado
contenido secundario
contenido complementario
6. Regla de foco editorial

Debe existir siempre:

→ un contenido principal

Evitar múltiples focos.

7. Agrupación editorial

El contenido debe agruparse por:

temática
contexto
relación
8. Flujo de lectura

El sistema debe guiar al usuario:

qué ver primero
qué ver después
qué puede ignorar
9. Escaneabilidad editorial

El usuario debe poder:

entender la portada en segundos
identificar lo importante
decidir rápido
10. Progresión de contenido

El sistema debe presentar:

lo esencial
lo relevante
lo ampliado
11. Densidad editorial

La cantidad de contenido visible debe estar controlada.

Evitar:

saturación
ruido
12. Modularidad editorial

El contenido se organiza en bloques reutilizables.

13. Relación entre bloques

Los bloques deben:

conectarse
no competir
14. Ritmo editorial

La combinación de bloques debe generar:

fluidez
alternancia
claridad
15. Integración de publicidad

La publicidad es parte del sistema editorial.

Debe:

integrarse
respetar jerarquía
no romper flujo
16. Contexto editorial

La jerarquía puede cambiar según:

actualidad
contexto
usuario
17. Sistema dinámico

El sistema debe poder:

reorganizar contenido
adaptarse a cambios
mantener coherencia
18. Evitar arbitrariedad editorial

No decidir:

por estética
por costumbre

Decidir por:

→ lógica informativa

19. Evaluación editorial

El sistema debe permitir evaluar:

claridad
jerarquía
comprensión
20. Regla final

Si el usuario no entiende qué es importante en segundos:

→ el sistema ha fallado

Ejemplo simplificado

Portada:

noticia principal → foco
secundarias → apoyo
bloques → contexto

↓

Resultado:

lectura clara

Cierre

El diseño editorial no es decoración.

Es organización de información.
