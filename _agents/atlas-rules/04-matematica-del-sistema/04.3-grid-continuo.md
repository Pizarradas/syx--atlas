04.3 — Grid continuo y sistema de layout (SYX Atlas)
Naturaleza de este documento

Este documento define cómo se construyen:

layouts
columnas
distribución de contenido
estructura de página

sin depender de:

viewports fijos
breakpoints arbitrarios
versiones separadas (mobile / desktop)
Principio base

No existen layouts distintos.
Existe un único sistema que se adapta.

1. El concepto de grid continuo

El sistema no utiliza grids rígidos por dispositivo.

Utiliza un grid que:

escala
se adapta
mantiene proporciones
2. Eliminación de breakpoints como base conceptual

Los breakpoints no son el sistema.

Son:

→ ajustes puntuales
→ no el núcleo del diseño

El sistema debe funcionar sin depender de ellos.

3. Unidad estructural

Todo layout se construye a partir de unidades proporcionales.

Ejemplo:

unidad base → 1
doble → 2
triple → 3
4. Sistema de columnas flexible

Las columnas no son fijas.

Se comportan como:

relaciones proporcionales
no como estructuras rígidas

Ejemplo:

2:1
3:1
1:1
5. Regla de continuidad

El layout debe transformarse de forma fluida.

Evitar:

saltos bruscos
reorganizaciones radicales
6. Prioridad de contenido

El layout se organiza según:

importancia
contexto
flujo de lectura

No según:

simetría
estética
7. Colapso progresivo

En contextos restrictivos:

el layout no se rompe
se reorganiza progresivamente

Ejemplo:

columnas → se apilan
bloques → se simplifican
8. Sistema de anchura

La anchura no se define en valores absolutos.

Se define en:

proporciones
relaciones
9. Sistema de márgenes

Los márgenes deben:

escalar con el sistema
no ser valores fijos
10. Sistema de contenedores

Los contenedores no son cajas rígidas.

Son:

estructuras adaptativas
con límites proporcionales
11. Regla de legibilidad

El layout debe respetar:

anchura óptima de lectura
longitud de línea adecuada
12. Densidad estructural

El layout regula:

cantidad de contenido visible
distribución de información
13. Relación con spacing

El grid y el spacing deben estar conectados.

No pueden funcionar como sistemas separados.

14. Modularidad

El layout se construye mediante bloques reutilizables.

15. Composición dinámica

El sistema debe permitir:

reorganizar contenido
adaptar estructuras
mantener coherencia
16. Regla de estabilidad

El layout no debe cambiar radicalmente entre contextos.

Debe evolucionar.

17. Integración de publicidad

La publicidad forma parte del grid.

Debe:

respetar proporciones
integrarse en la estructura
no romper el flujo
18. Sistema de alineación

La alineación debe ser:

consistente
estructural
no decorativa
19. Evaluación del layout

Cada layout debe cumplir:

coherencia
legibilidad
continuidad
escalabilidad
20. Regla final

Si el layout depende de un breakpoint para funcionar:

→ el sistema está mal definido

Ejemplo simplificado

Desktop:

columna principal → 2
columna secundaria → 1

↓

Tablet:

proporción se mantiene
distribución se ajusta

↓

Mobile:

columnas se apilan
jerarquía se mantiene
Cierre

El layout no se diseña para pantallas.

Se diseña como sistema continuo.
