04.4 — Densidad y carga cognitiva (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema gestiona:

la cantidad de información visible
la organización del contenido
el esfuerzo mental del usuario

No se trata de diseño visual.
Se trata de procesamiento cognitivo.

Principio base

El problema no es cuánta información hay,
sino cuánta información debe procesarse a la vez.

1. Qué es la carga cognitiva

Es el esfuerzo mental necesario para:

entender
decidir
navegar

El sistema debe minimizarla.

2. Tipos de carga cognitiva
Intrínseca

Proviene del contenido en sí.

No puede eliminarse.
Solo puede organizarse mejor.

Extrínseca

Proviene del diseño.

Debe reducirse al máximo.

Germana

Relacionada con el aprendizaje.

Debe facilitarse.

3. Principio de reducción

El sistema debe:

eliminar ruido
simplificar estructuras
evitar redundancia
4. Densidad de información

La densidad define cuánta información hay en un espacio.

Variables:

número de elementos
tamaño
spacing
5. Regla de equilibrio

El sistema debe evitar:

saturación (demasiado contenido)
vacío (falta de información útil)
6. Chunking (agrupación mental)

El usuario procesa información en bloques.

El sistema debe:

agrupar elementos relacionados
separar contextos distintos
7. Jerarquía cognitiva

El usuario debe identificar rápidamente:

qué es importante
qué es secundario
qué puede ignorar
8. Escaneabilidad

El contenido debe poder recorrerse sin leer todo.

Esto implica:

estructura clara
patrones repetidos
puntos de entrada visibles
9. Regla de foco

En cada momento debe existir:

→ un foco principal

Evitar múltiples focos simultáneos.

10. Progresión de información

La información debe revelarse en capas:

lo esencial
lo relevante
lo secundario
11. Reducción de decisiones

El sistema debe evitar obligar al usuario a decidir constantemente.

Menos decisiones:

→ mejor experiencia

12. Consistencia cognitiva

El sistema debe comportarse de forma predecible.

Esto reduce:

aprendizaje
esfuerzo mental
13. Ritmo cognitivo

El contenido debe alternar:

bloques densos
espacios de descanso
14. Evitar sobreestimulación

Demasiados estímulos generan:

fatiga
rechazo
abandono
15. Relación con el spacing

El spacing es la herramienta principal para:

reducir carga
organizar información
16. Relación con jerarquía

La jerarquía debe ser evidente sin esfuerzo.

17. Relación con layout

El layout debe facilitar:

lectura
navegación
comprensión
18. Sistema de límites

Definir:

máximo de elementos visibles por bloque
niveles jerárquicos permitidos
densidad máxima
19. Evaluación cognitiva

Cada solución debe responder a:

¿es fácil de entender?
¿es fácil de recorrer?
¿reduce esfuerzo mental?
20. Regla final

Si el usuario necesita pensar demasiado:

→ el sistema ha fallado

Ejemplo simplificado

Problema:

Demasiados elementos visibles

↓

Solución:

agrupar
reducir
jerarquizar

↓

Resultado:

Menor carga cognitiva

Cierre

Diseñar no es añadir.

Es decidir qué no mostrar.
