03 — Decision Engine (SYX Atlas)
Naturaleza de este documento

Este documento define cómo se toman decisiones dentro del sistema.

Es el puente entre:

principios → (01)
modelo mental → (02)
ejecución → (siguientes niveles)
Qué es el Decision Engine

El Decision Engine es el mecanismo mediante el cual:

un problema → se traduce en reglas
unas reglas → se convierten en estructura
una estructura → genera una solución
Principio base

No se resuelven problemas directamente.
Se resuelven mediante reglas que eliminan el problema.

Flujo de decisión obligatorio

Claude debe seguir siempre este flujo:

1. Entrada (input)

Identificar:

tipo de contenido
objetivo
restricciones
contexto de uso

Ejemplo:

noticia
portada
módulo editorial
bloque publicitario
2. Definición del problema

Transformar la necesidad en problema estructural:

❌ “hacerlo más visual”
✅ “mejorar jerarquía y escaneabilidad”

3. Activación de principios

Seleccionar principios relevantes:

carga cognitiva
jerarquía
continuidad
proporción
consistencia
4. Traducción a variables

Convertir el problema en variables medibles:

tamaño
proporción
spacing
densidad
ritmo
agrupación
5. Definición de reglas

Establecer reglas claras:

Ejemplo:

relación de tamaños (1:1.25, 1:1.5…)
spacing basado en escala
número máximo de elementos por bloque
niveles de jerarquía
6. Construcción estructural

Definir:

layout
bloques
distribución
flujo

Sin pensar en estética.

7. Evaluación del sistema

Validar:

coherencia
simplicidad
reutilización
ausencia de excepciones
8. Resultado

La forma emerge como consecuencia.

Tipos de decisiones

El sistema trabaja con tres niveles:

Nivel 1 — Decisiones estructurales

Definen:

layout
jerarquía
organización

Son las más importantes.

Nivel 2 — Decisiones relacionales

Definen:

proporciones
spacing
ritmo

Conectan los elementos.

Nivel 3 — Decisiones superficiales

Definen:

color
estilo
detalles visuales

Son consecuencia, no origen.

Regla de prioridad

Siempre se resuelve en este orden:

estructura
relaciones
superficie

Nunca al revés.

Sistema de variables

Todo problema debe traducirse a variables controlables:

espacio
tamaño
proporción
densidad
repetición
ritmo

Si algo no puede expresarse como variable, no está bien definido.

Sistema de restricciones

Las restricciones no son un problema.

Son parte del sistema.

Claude debe:

trabajar con ellas
no evitarlas
no forzarlas
Eliminación de arbitrariedad

Toda decisión debe responder a:

una regla
una relación
una necesidad estructural

Nunca a:

preferencia
estética
costumbre
Sistema de validación

Cada solución debe cumplir:

coherencia con principios
consistencia interna
simplicidad
capacidad de reutilización
Sistema de errores

Si una solución:

requiere excepciones
rompe consistencia
no escala

Entonces:

→ no es válida
→ debe replantearse

Principio de simplificación

Ante múltiples soluciones:

→ elegir siempre la más simple
→ que mantenga coherencia

Principio de no fragmentación

Evitar:

soluciones específicas para cada caso
lógica diferente según contexto

Buscar:

→ una única regla que cubra múltiples escenarios

Principio de crecimiento controlado

El sistema no crece añadiendo más reglas.

Crece:

refinando las existentes
eliminando redundancias
unificando soluciones
Principio de trazabilidad

Cada decisión debe poder rastrearse:

problema → variable → regla → solución
Ejemplo simplificado

Problema:
“El contenido no se entiende rápido”

↓

Variables:

densidad
jerarquía
spacing

↓

Reglas:

máximo 3 niveles jerárquicos visibles
spacing proporcional a jerarquía
agrupación por bloques

↓

Resultado:
Estructura clara y escaneable

Relación con el sistema

El Decision Engine:

no diseña
no implementa

Define cómo decidir correctamente.

Regla de precedencia

En caso de conflicto:

propósito (00)
principios (01)
modelo mental (02)
decision engine (03)
resto
Cierre

El valor del sistema no está en lo que construye,
sino en cómo decide construirlo.
