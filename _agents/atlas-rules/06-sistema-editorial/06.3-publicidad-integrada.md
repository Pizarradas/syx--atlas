06.3 — Publicidad integrada en el sistema (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema integra:

bloques publicitarios
contenido patrocinado
formatos comerciales

como parte estructural del sistema editorial.

Principio base

La publicidad no es un elemento externo.
Es parte del sistema.

1. Problema habitual

En la mayoría de sistemas:

la publicidad se inserta sin lógica
rompe el layout
interrumpe el flujo
genera rechazo
2. Enfoque de SYX Atlas

La publicidad se trata como:

un bloque más del sistema
con reglas
con proporciones
con integración estructural
3. Tipos de publicidad y sus dimensiones proporcionales

El sistema contempla cuatro tipos con dimensiones definidas en relación al grid:

| Tipo | Formato estándar | Posición en el sistema |
|---|---|---|
| **Sidebar** | 300×250 (1:1.2) | Columna secundaria del grid editorial (sidebar) |
| **Billboard** | 970×90 (1:0.09) | Entre zonas de contenido. Nunca dentro de una zona. |
| **Mobile banner** | 320×50 (1:0.16) | Solo en mobile. Debajo del ticker o entre zonas. |
| **Contenido patrocinado** | Mismo layout que card N3 | Se marca con etiqueta "Publicidad". No interrumpe el flujo. |

**Regla de dimensiones**: los formatos no se definen en px absolutos en el layout. El bloque `.mol-publicidad` usa `aspect-ratio` y ocupa el espacio proporcional que le corresponde en el grid.

4. Nivel de jerarquía publicitaria

La publicidad no tiene nivel N1, N2 ni N3 editorial.

Tiene su propio nivel: **NP (Nivel Publicitario)**.

Reglas del nivel NP:
- Nunca mayor presencia visual que el contenido N2
- En sidebar: comparte columna con datos de mercado; no compite con el contenido principal
- En billboard: es zona de pausa entre secciones; no interrumpe una zona activa

5. Posiciones permitidas y prohibidas

**Posiciones permitidas**:
- Sidebar de la zona de desarrollo (junto a noticias destacadas)
- Entre la zona de desarrollo y la zona de profundidad
- Entre la zona de profundidad y la zona de análisis
- Al final de la zona de análisis (antes del footer)

**Posiciones prohibidas**:
- Dentro del hero principal (zona N1)
- Entre dos noticias de la misma zona (rompe flujo)
- Encima o debajo del titular de cualquier noticia N2
- Sticky sobre el contenido (salvo formatos acordados con negocio, con scroll-dismiss)

6. Regla de no ruptura

La publicidad no debe:

romper el flujo
alterar la jerarquía
generar saltos bruscos en el layout

**Implementación**: el bloque publicitario es un hijo del grid como cualquier otro bloque. `position: absolute` o `position: fixed` sin contenedor previsto en el grid son violaciones del contrato CP-01 (ver `09.0`).

7. Densidad publicitaria máxima

| Zona | Máximo bloques publicitarios |
|---|---|
| Hero (N1) | 0 |
| Zona de desarrollo | 1 (sidebar) |
| Zona de profundidad | 1 (billboard entre zonas) |
| Zona de análisis | 1 (billboard al final) |
| **Total por portada** | **3** |

**Regla**: dos bloques publicitarios contiguos sin contenido editorial entre ellos son violación de densidad.
10. Claridad

El usuario debe identificar claramente:

→ qué es contenido
→ qué es publicidad

11. Integración visual

La publicidad debe:

integrarse en el sistema
sin camuflarse
sin engañar
12. Regla de respeto al usuario

La publicidad no debe:

dificultar la lectura
generar fricción
obligar a interactuar
13. Publicidad como bloque modular

Debe comportarse como:

cualquier otro módulo
reutilizable
escalable
14. Adaptación al contexto

En mobile:

reducir impacto
simplificar formato

En contextos extremos:

→ mínima intrusión

15. Publicidad dinámica

El sistema debe permitir:

variar formatos
adaptarse a campañas
mantener coherencia
16. Señales de fallo
interrumpe lectura
rompe layout
genera rechazo
17. Señales de integración correcta
fluye con el contenido
no distrae en exceso
respeta jerarquía
18. Relación con negocio

El sistema debe:

permitir monetización
sin degradar experiencia
19. Evaluación publicitaria

Preguntas clave:

¿rompe el flujo?
¿interrumpe la lectura?
¿respeta el sistema?
20. Regla final

Si la publicidad rompe el sistema:

→ el sistema está mal diseñado

Ejemplo simplificado

Bloque editorial

↓

espacio natural

↓

bloque publicitario integrado

↓

continúa contenido

↓

Resultado:

flujo mantenido

Cierre

La publicidad no debe adaptarse al sistema.

El sistema debe estar preparado para integrarla.

Lectura correcta de este documento

Este documento define cómo integrar publicidad sin romper la experiencia.
