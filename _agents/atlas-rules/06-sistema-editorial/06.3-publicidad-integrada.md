06.3 — Publicidad integrada en el sistema (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema integra:

bloques publicitarios
contenido patrocinado
formatos comerciales

como parte estructural del sistema editorial.

Principio base

La publicidad no es un elemento externo.
Es parte del sistema.

1. Problema habitual

En la mayoría de sistemas:

la publicidad se inserta sin lógica
rompe el layout
interrumpe el flujo
genera rechazo
2. Enfoque de SYX Atlas

La publicidad se trata como:

un bloque más del sistema
con reglas
con proporciones
con integración estructural
3. Tipos de publicidad

El sistema debe contemplar:

display (banners)
contenido patrocinado
módulos integrados
formatos especiales
4. Nivel de jerarquía publicitaria

La publicidad tiene jerarquía propia, pero:

nunca debe competir con el contenido principal
debe integrarse en la jerarquía global
5. Regla de integración

Todo bloque publicitario debe:

respetar el grid
respetar el spacing
respetar proporciones
6. Regla de no ruptura

La publicidad no debe:

romper el flujo
alterar la jerarquía
generar saltos bruscos
7. Ubicación estratégica

La publicidad debe colocarse en:

puntos naturales de pausa
transiciones entre bloques
zonas de menor carga cognitiva
8. Ritmo publicitario

La publicidad debe seguir el ritmo del sistema.

Evitar:

acumulación
repetición excesiva
9. Densidad publicitaria

Debe existir un equilibrio entre:

contenido
publicidad
10. Claridad

El usuario debe identificar claramente:

→ qué es contenido
→ qué es publicidad

11. Integración visual

La publicidad debe:

integrarse en el sistema
sin camuflarse
sin engañar
12. Regla de respeto al usuario

La publicidad no debe:

dificultar la lectura
generar fricción
obligar a interactuar
13. Publicidad como bloque modular

Debe comportarse como:

cualquier otro módulo
reutilizable
escalable
14. Adaptación al contexto

En mobile:

reducir impacto
simplificar formato

En contextos extremos:

→ mínima intrusión

15. Publicidad dinámica

El sistema debe permitir:

variar formatos
adaptarse a campañas
mantener coherencia
16. Señales de fallo
interrumpe lectura
rompe layout
genera rechazo
17. Señales de integración correcta
fluye con el contenido
no distrae en exceso
respeta jerarquía
18. Relación con negocio

El sistema debe:

permitir monetización
sin degradar experiencia
19. Evaluación publicitaria

Preguntas clave:

¿rompe el flujo?
¿interrumpe la lectura?
¿respeta el sistema?
20. Regla final

Si la publicidad rompe el sistema:

→ el sistema está mal diseñado

Ejemplo simplificado

Bloque editorial

↓

espacio natural

↓

bloque publicitario integrado

↓

continúa contenido

↓

Resultado:

flujo mantenido

Cierre

La publicidad no debe adaptarse al sistema.

El sistema debe estar preparado para integrarla.
