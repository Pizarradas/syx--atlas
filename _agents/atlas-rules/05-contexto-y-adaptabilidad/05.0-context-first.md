05 — Contexto y adaptabilidad (SYX Atlas)
Naturaleza de este documento

Este documento define cómo el sistema se adapta a:

condiciones reales de uso
entornos cambiantes
dispositivos actuales y futuros

No se diseña para pantallas.
Se diseña para situaciones.

Principio base

El diseño no depende del dispositivo.
Depende del contexto en el que se usa.

1. Qué es el contexto

El contexto incluye:

dispositivo
entorno (luz, ruido, movimiento)
tiempo disponible
estado de atención
tipo de interacción
2. Fin del enfoque viewport-based

El sistema no se construye en base a:

mobile
tablet
desktop

Estos son solo casos particulares, no la base.

3. Context-first

El sistema se diseña desde el peor escenario posible:

pantalla pequeña
atención fragmentada
interacción limitada
condiciones imperfectas

Si funciona ahí:

→ funcionará en cualquier otro contexto

4. Continuidad contextual

El sistema no cambia entre contextos.

Se adapta de forma continua.

Evitar:

versiones distintas
soluciones paralelas
lógica duplicada
5. Variables de contexto

El sistema debe responder a variables como:

tamaño disponible
densidad de información
tipo de input (táctil, mouse, voz)
velocidad de uso
nivel de atención
6. Interacción adaptativa

El sistema debe adaptarse a:

tacto (zonas grandes, precisión baja)
ratón (precisión alta)
voz (sin interfaz visual directa)
7. Atención como recurso limitado

El sistema debe asumir:

el usuario no está concentrado
el tiempo es limitado
la interacción es rápida
8. Simplificación en contexto restrictivo

En condiciones adversas:

reducir elementos
simplificar estructura
priorizar contenido clave
9. Progresión contextual

El sistema debe mostrar:

lo esencial primero
lo secundario después
lo opcional al final
10. Diseño para interrupción

El usuario puede:

entrar
salir
volver

El sistema debe permitir:

→ reenganche inmediato

11. Contextos futuros

El sistema debe ser válido para:

coche (atención mínima)
reloj (espacio extremo)
dispositivos aún no definidos
12. Regla de independencia tecnológica

El sistema no depende de:

resoluciones actuales
dispositivos concretos
13. Adaptación sin ruptura

El sistema no debe:

romper layout
cambiar lógica
perder coherencia
14. Densidad contextual

La cantidad de información visible debe adaptarse al contexto.

15. Jerarquía contextual

Lo importante cambia según el contexto.

El sistema debe poder priorizar dinámicamente.

16. Tiempo como variable

El sistema debe considerar:

uso rápido
uso profundo
17. Sistema resiliente

El sistema debe funcionar incluso en:

malas condiciones de red
baja calidad de pantalla
interacción imprecisa
18. Regla de degradación elegante

Si el sistema no puede mostrar todo:

→ debe simplificar sin romperse

19. Evaluación contextual

Cada solución debe evaluarse en:

mobile real
uso rápido
condiciones imperfectas
20. Regla final

Si el sistema depende de un dispositivo específico:

→ no es un sistema válido

Ejemplo simplificado

Contexto:

usuario en móvil, poco tiempo

↓

Sistema:

reduce densidad
simplifica layout
prioriza titulares

↓

Resultado:

uso eficiente

Cierre

El contexto no es una variable más.

Es la base sobre la que se construye todo.
