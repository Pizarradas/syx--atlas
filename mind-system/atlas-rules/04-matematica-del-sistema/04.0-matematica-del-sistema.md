04 — Marco matemático del sistema (SYX Atlas)
Naturaleza de este documento

Este documento define las bases matemáticas que sustentan el sistema.

No es una guía técnica.
Es el marco que permite que todas las decisiones sean:

medibles
coherentes
reproducibles
Principio fundamental

Todo en el sistema debe poder expresarse como una relación.

No se trabaja con valores absolutos.
Se trabaja con:

proporciones
escalas
relaciones
1. Sistema de proporciones

El sistema se basa en relaciones proporcionales entre elementos.

Nunca en tamaños arbitrarios.

Tipos de proporción
proporciones simples (1:1, 1:2)
proporciones progresivas (1:1.25, 1:1.5, 1:1.618)
proporciones jerárquicas
2. Escalas

Las escalas definen cómo crecen los elementos dentro del sistema.

Tipos de escala
escala tipográfica
escala espacial
escala de componentes
Regla

Toda escala debe:

ser consistente
ser continua
permitir crecimiento controlado
3. Ritmo

El ritmo define la repetición y organización del espacio.

Tipos de ritmo
vertical (flujo de lectura)
horizontal (agrupación)
modular (bloques)
Regla

El ritmo debe ser:

perceptible
consistente
no arbitrario
4. Sistema de spacing

El espacio no es decorativo.

Es estructural.

Regla base

Todo spacing debe derivar de una escala.

Evitar:

valores sueltos
ajustes manuales
5. Densidad

La densidad define cuánta información hay en un espacio.

Variables
número de elementos
tamaño relativo
espacio disponible
Regla

Debe mantenerse dentro de límites controlados para evitar:

saturación
vacío excesivo
6. Agrupación

Los elementos se organizan por proximidad y relación.

Principios
elementos cercanos → relacionados
elementos separados → independientes

Esto no es visual.
Es matemático.

7. Jerarquía proporcional

La jerarquía se define mediante relaciones de tamaño y espacio.

No mediante estilos arbitrarios.

8. Continuidad matemática

El sistema debe comportarse de forma continua.

Evitar:

saltos bruscos
cambios no progresivos
9. Sistema de límites

Todo sistema necesita límites.

Ejemplos
máximo de elementos por bloque
mínimo de tamaño legible
máximo de densidad
10. Escalabilidad matemática

El sistema debe funcionar en cualquier tamaño sin romper relaciones.

11. Eliminación de valores absolutos

Evitar:

px fijos como base conceptual
tamaños aislados

Usar:

relaciones
escalas
proporciones
12. Sistema de módulos

Los bloques deben construirse como unidades proporcionales.

13. Repetición controlada

La repetición genera coherencia.

Pero debe:

estar regulada
no ser excesiva
14. Variación dentro del sistema

La variación no es libre.

Debe estar controlada por reglas.

15. Coherencia matemática global

Todas las partes del sistema deben responder a las mismas reglas matemáticas.

16. Principio de precisión

Pequeñas desviaciones generan incoherencias acumuladas.

El sistema debe ser preciso.

17. Principio de reducción

Menos relaciones → más control
Más relaciones → más complejidad

El sistema debe buscar equilibrio.

18. Relación con la percepción

Las matemáticas deben alinearse con cómo el usuario percibe:

tamaño
distancia
jerarquía
19. Medición del sistema

Todo debe poder medirse:

spacing
proporciones
densidad
ritmo
20. Regla final

Si una decisión no puede expresarse matemáticamente:

→ no pertenece al sistema

Cierre

Las matemáticas no son una herramienta.

Son el lenguaje del sistema.
