# 08.0 — Registro de tokens (SYX Atlas)

## Naturaleza de este documento

Este documento define la arquitectura de tokens del sistema y su inventario.

No es una guía de uso.
Es el contrato de referencia que toda implementación debe consultar.

---

## Principio base

> Todo valor repetible debe existir como token o derivarse de uno.  
> Ningún componente usa valores primitivos directamente.

---

## 1. Jerarquía obligatoria

```
Primitive → Semantic → Component
```

- **Primitive**: valor crudo (hex, número, rem). Solo existe en la capa de tokens.
- **Semantic**: mapeo con significado funcional (`--color-text-primary`, `--space-md`).
- **Component**: token específico de un componente, derivado de semánticos.

Nunca saltar niveles.

## 6. Tokens de componente

Los tokens de componente se derivan de semánticos y se nombran así:

```
--[componente]-[propiedad]: var(--semantic-[token]);
```

**Ejemplo**:

```css
--card-bg: var(--semantic-color-surface);
--card-border: var(--semantic-color-border);
--card-title-size: var(--semantic-type-title);
--card-spacing: var(--semantic-space-md);
```

**Regla**: si un componente necesita un valor que no existe como semántico, primero se crea el semántico, luego el de componente.

## 9. Reglas de uso

1. **Nunca usar un primitivo en un componente.** Siempre a través de semántico.
2. **Nunca hardcodear hex, px o rem arbitrarios.** Todo sale de primitivos.
3. **Nunca crear un semántico sin primitivo registrado aquí.**
4. **Antes de crear un token nuevo:** comprobar si ya existe uno que sirva.
5. **Si falta un token:** añadirlo en este documento antes de usarlo.

---

## 10. Proceso de adición de token

```
1. ¿Existe ya un token que cumpla esta función?
   → Sí: usarlo
   → No: continuar

2. ¿Es un valor primitivo nuevo?
   → Añadirlo a la sección correspondiente de este documento
   → Nombrarlo con el patrón --primitive-[categoría]-[descripción]

3. ¿Necesita un semántico?
   → Añadirlo al mapeo de semánticos
   → Nombrarlo con el patrón --semantic-[categoría]-[función]

4. ¿Es específico de un componente?
   → Declararlo en el componente, derivando del semántico
```

## Cierre

Este documento es el único inventario autorizado de tokens del sistema.

Si un valor no está aquí, no pertenece al sistema.

Si pertenece al sistema, debe estar aquí.
