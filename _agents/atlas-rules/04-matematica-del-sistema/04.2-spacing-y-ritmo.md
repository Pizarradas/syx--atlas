04.2 — Spacing y ritmo (SYX Atlas)
Naturaleza de este documento

Este documento define cómo se construyen:

el flujo de lectura
la separación entre elementos
la organización del contenido
la percepción de orden

El spacing no es decoración.
Es estructura cognitiva.

Principio base

El usuario no lee interfaces.
Las recorre.

El ritmo define cómo se recorre el contenido.

1. El espacio como sistema

El espacio no se aplica.

Se construye mediante reglas.

Debe responder a:

escala definida (04.1)
jerarquía
agrupación
2. Tipos de spacing

El sistema distingue tres niveles:

Nivel 1 — Micro spacing

Define relaciones internas:

entre líneas
entre icono y texto
entre elementos pequeños

Debe ser:

sutil
consistente
casi invisible
Nivel 2 — Meso spacing

Define agrupaciones:

entre bloques relacionados
dentro de módulos
entre elementos de un mismo contexto

Debe indicar:

→ relación directa

Nivel 3 — Macro spacing

Define separación estructural:

entre secciones
entre bloques principales
entre zonas editoriales

Debe indicar:

→ cambio de contexto

3. Regla de jerarquía espacial

A mayor importancia:

→ mayor espacio alrededor

Esto refuerza:

visibilidad
prioridad
comprensión
4. Regla de agrupación

Elementos relacionados deben estar más cerca entre sí
que de otros elementos.

Esto construye:

bloques mentales
unidades de información
5. Ritmo vertical (clave en editorial)

El ritmo vertical define cómo se lee la página.

Debe ser:

consistente
predecible
fluido
6. Ritmo horizontal

Menos relevante que el vertical, pero importante para:

agrupación visual
comparaciones
navegación
7. Sistema de repetición

El ritmo se construye mediante repetición controlada:

spacing consistente
patrones repetidos
bloques reconocibles
8. Evitar irregularidad

El sistema debe evitar:

espacios incoherentes
saltos arbitrarios
desalineaciones
9. Relación con escala

Todo spacing debe derivar de la escala base.

Nunca debe ser independiente.

10. Densidad controlada

El spacing regula la densidad.

Demasiado poco espacio:

→ saturación

Demasiado espacio:

→ pérdida de continuidad

11. Flujo de lectura

El spacing debe facilitar:

escaneo rápido
comprensión jerárquica
navegación intuitiva
12. Separación vs conexión

El espacio tiene dos funciones:

separar
conectar

Debe usarse con intención.

13. Ritmo modular

Los bloques deben mantener un ritmo coherente:

alturas proporcionales
separaciones consistentes
repetición estructural
14. Regla de consistencia

El mismo tipo de relación debe tener el mismo spacing.

Siempre.

15. Ruptura controlada

El sistema puede romper el ritmo, pero solo para:

destacar
cambiar contexto
generar énfasis

Nunca por accidente.

16. Ritmo y percepción

El usuario percibe ritmo como:

orden
claridad
profesionalidad
17. Ritmo y velocidad de lectura

Un buen ritmo:

acelera la lectura
reduce esfuerzo
mejora comprensión
18. Sistema de límites

Definir:

spacing mínimo
spacing máximo
densidad máxima
19. Evaluación del ritmo

Cada layout debe evaluarse según:

fluidez
coherencia
claridad
20. Regla final

Si el spacing no responde a una regla:

→ es ruido

Ejemplo simplificado

Jerarquía:

título → espacio grande
subtítulo → espacio medio
texto → espacio base

Agrupación:

elementos relacionados → cercanos
bloques distintos → separados
Cierre

El ritmo no se diseña.

Se construye.
