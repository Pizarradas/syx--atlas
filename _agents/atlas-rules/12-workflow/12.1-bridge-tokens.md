# 12.1 — Bridge de tokens: Atlas → SYX Modes

## Naturaleza de este documento

Cuando Atlas invoca a TOKEN mode o UI mode, los nombres conceptuales de Atlas deben traducirse a los nombres técnicos autorizados del sistema SYX.

Este documento es la tabla de traducción de referencia.

---

## Principio base

> Los modes son la autoridad técnica en naming de tokens.
> Atlas es la autoridad conceptual en su función.
> Este bridge convierte conceptos en nombres.

---

## 1. Color — superficie y texto

| Concepto Atlas | Token SYX Modes (autorizado) | Notas |
|---|---|---|
| Fondo de página | `--semantic-color-bg-primary` | Base de toda página |
| Fondo de tarjeta / panel | `--semantic-color-bg-secondary` | Superficies elevadas |
| Fondo en hover de superficie | `--semantic-color-bg-tertiary` | Estados interactivos sobre bg |
| Color de acción / marca | `--semantic-color-primary` | Botones, enlaces, CTAs |
| Hover del color de acción | `--semantic-color-primary-hover` | Estado hover de primario |
| Fondo oscuro (cabecera, footer) | `--semantic-color-bg-inverse` | Superficies oscuras de marca |
| Texto de mayor jerarquía | `--semantic-color-text-primary` | Titulares N1–N3 |
| Texto de apoyo | `--semantic-color-text-secondary` | Entradillas, subtítulos |
| Texto de metadatos | `--semantic-color-text-tertiary` | Fechas, autores, labels |
| Texto sobre fondo oscuro | `--semantic-color-text-inverse` | Texto blanco sobre header/footer |
| Separador suave | `--semantic-color-border-subtle` | Divisores entre cards |
| Separador con peso | `--semantic-color-border-default` | Bordes de componentes |
| Separador fuerte | `--semantic-color-border-strong` | Énfasis estructural |

## 2. Color — datos de mercado (editorial específico)

| Concepto Atlas | Token SYX Modes | Notas |
|---|---|---|
| Valor positivo (alza) | `--semantic-color-state-success` | Verde. Texto sobre fondo claro. |
| Fondo de valor positivo | `--semantic-color-state-success-subtle` | Verde muy claro. Badge bg. |
| Valor negativo (baja) | `--semantic-color-state-error` | Rojo. Texto sobre fondo claro. |
| Fondo de valor negativo | `--semantic-color-state-error-subtle` | Rojo muy claro. Badge bg. |
| Sin variación (neutro) | `--semantic-color-text-tertiary` | Gris estándar de metadatos. |

**Nota sobre ticker en fondo oscuro**: en el ticker sobre cabecera oscura, los colores alza/baja se ajustan al contexto. Son variantes de `--semantic-color-state-success/error` con mayor luminosidad para mantener contraste.

## 3. Espaciado — mapping nivel Atlas → token SYX

Atlas define el spacing por su función editorial. SYX lo nombra por tipo de uso (inset, stack, inline).

| Función Atlas | Token SYX Modes | Valor |
|---|---|---|
| Gap micro (icono/texto) | `--semantic-space-inset-xs` | 4px |
| Separación interna de componente | `--semantic-space-inset-sm` | 8px |
| Separación estándar en bloque | `--semantic-space-inset-md` | 16px |
| Separación generosa en bloque | `--semantic-space-inset-lg` | 24px |
| Separación entre bloques N3 | `--semantic-space-stack-md` | 16px |
| Separación entre bloques N2 | `--semantic-space-stack-lg` | 24px |
| Separación entre zonas N1/N2 | `--semantic-space-stack-xl` | 32px |
| Separación entre secciones principales | `--semantic-space-stack-2xl` | 48px |
| Padding de página (container) | `--semantic-space-inset-md/lg/xl` | Escala con breakpoint |

## 5. Naming de clases — Atlas y SYX son compatibles

Las clases HTML usan el mismo patrón en ambos sistemas. No hay traducción necesaria.

| Capa | Prefijo | Ejemplo |
|---|---|---|
| Atom | `.atom-` | `.atom-headline--xl`, `.atom-etiqueta` |
| Molecule | `.mol-` | `.mol-card-noticia`, `.mol-publicidad` |
| Organism | `.org-` | `.org-hero-principal`, `.org-grid-editorial` |
| Modificador | `--` | `--destacada`, `--secundaria`, `--alza` |
| Elemento | `__` | `__titular`, `__meta`, `__cta` |

---

## 6. Formato de color — contexto de uso

| Contexto | Formato |
|---|---|
| Implementación SCSS compilada (modes) | OKLCH obligatorio en primitivos |
| HTML/CSS standalone de Atlas (POCs, portadas demo) | Hex permitido, no se compila a SCSS |
| Tablas de contraste y documentación | Hex como referencia legible |

**Conversión pendiente**: los primitivos hex de `atlas-rules/08-tokens/08.0-registro-tokens.md` deben convertirse a OKLCH cuando se promuevan a implementación SCSS. Esa conversión es responsabilidad de TOKEN mode.

---

## 7. Cómo usar este documento al invocar TOKEN mode

Incluir en la petición:

```
[SYX: TOKEN]: [descripción de tokens necesarios]

Equivalencias de naming Atlas → SYX para esta petición:
- "fondo de página" → --semantic-color-bg-primary
- "texto de titular N2" → --semantic-color-text-primary
- "separación entre cards N2" → --semantic-space-stack-lg
Ver atlas-rules/12-workflow/12.1-bridge-tokens.md para referencia completa.
```
